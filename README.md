# 小资金商品期货·模拟盘跟踪系统

一个纯静态的网页版商品期货模拟盘跟踪系统：观察池 → 基本面 → 三因子信号 → 模拟交易 → 资金曲线。

- 起始资金：¥15,000（可调）
- 目标：¥1,000,000
- 数据保存在浏览器 `localStorage`，敏感信息（Token）走 `sessionStorage`
- 支持一键同步到 GitHub Gist 云备份

## 在线访问

https://lee20260315.github.io/futures-tracker/

## 本地运行

直接用浏览器打开 `index.html`，或运行本地静态服务器：

```
python -m http.server 8000
```

然后访问 http://localhost:8000/

## 页面结构

## 主要功能

### 1. 品种观察池
- 38 个商品期货品种，覆盖国内四大交易所（SHFE/DCE/CZCE/GFEX）+ 中金所（CFFEX）
- **按交易所分组展示**：SHFE（上海期货）/ DCE（大连商品）/ CZCE（郑州商品）/ GFEX（广州期货）四组，组标题为"交易所全称+简称"格式
- **核心层/观察层两级标记**：
  - 核心层（9 个：棕榈油/白糖/棉花/铜/黄金/白银/天然橡胶/多晶硅/碳酸锂）走完整五维基本面打分，橙色标签
  - 观察层（豆油/菜油及现有其他预置品种）仅显示现价+分位%，不进基本面评分，灰色标签
- 自动维护现价、历史分位（基于近 3 年价格区间）、成本线（含生产成本参考回填）、价差/价差%
- **成本线空值占位**：既无手动值也无参考值时显示可点击"+设置成本线"占位，点击 prompt 录入后价差/价差%自动重算
- **分位%显示**：有历史区间的品种显示数值+mini-bar；price-history.json 未收录的品种显示"暂无历史"（非空白）
- 支持手动添加/删除品种，合约代码校验（格式/前缀/过期/交割月）

### 2. 基本面跟踪
- 每品种 5 维度评分（供需格局 / 库存水平 / 基差结构 / 宏观环境 / 技术形态，0-10 分）
- 按数据真实来源分档填充：外部飞书日报解析出 active 指标后，按 `indicatorToDimension` 前缀映射归入对应维度并加权出分；无来源维度留空标注"无数据源"/"无品种级数据源"
- 品种模板（`shared/fund-dimension-config.js`）：
  - `agri`（农产品/新能源：棕榈油/天然橡胶/白糖/棉花/碳酸锂/多晶硅）：供需/库存/宏观按日报填充，基差/技术形态待手动输入
  - `metal`（贵金属/铜）：仅宏观情绪分有效，其余四维标灰"无品种级数据源"
  - `lightweight`（螺纹钢/玻璃）：仓单分位展示，不计算归一化分
  - `observe`（观察层：豆油/菜油及非核心品种）：五维全"—"，hint 显示"仅行情观察，未纳入基本面评分"
- **位置定性小结**：每个核心层品种五维评分下方新增一句话小结区，将三数据源翻译为可决策结论：
  - 估值位置 = 分位%（低位→赔率好）；三级回退读数：① pool.percentile（观察池已算）→ ② computePercentile(symbol, pool.price) 实时算 → ③ priceHistory.avg 均值参考（基本面页未刷行情时用）；标"（实时）"/"（均值参考）"区分来源
  - 筹码位置 = COT 持仓拥挤度（拥挤→反噬风险，缺失标"数据不足"）
  - 现货强弱 = 基差维度评分（深贴水→现货强、多头有支撑）
  - 综合结论：good≥2 → "赔率占优，可关注"；bad≥2 → "风险升高，观望"；muted≥2 → "数据不足，观望"；其余 → "中性，观望"
  - 数据缺失维度如实标"数据不足"，不硬凑结论；纯展示层，不影响综合评分逻辑
- **拒绝出分闸门（全局统一，不因品种类别豁免）**：双闸门判定，任一触发即拒绝出分：
  - 主判定：有效维度**个数**占比 < 50%（如 5 维中仅 2 维有效=40% → 拒绝；与"5维中2维有效"的直观口径一致）
  - 辅判定：有效维度**权重**占比 < 50%（保留原逻辑兜底）
  - 拒绝时综合评分栏显示"数据不足·拒绝出分"并列出缺失项 + 有效维度占比；agri 模板额外显示有效维度摘要（如"供需格局库存水平有效，待补宏观环境基差结构技术形态"），metal 模板附宏观情绪分
  - 修复历史 bug：原逻辑只看权重占比，agri 模板 supply(0.35)+inventory(0.30)=0.65>0.5 不触发拒绝，导致白糖仅 2 维有效却显示 94.6/100 虚高分；改为维度个数占比主导后，白糖/碳酸锂/白银三类品种判定一致
- **空状态模板**：未选品种时五维显示"—"、综合评分显示"— / 100"，不再出现 0/50 旧刻度
- 配置共享模块 `shared/fund-dimension-config.js`：维度/指标映射/品种模板/权重/拒绝阈值，含开发态 `validateConfig` 校验

### 3. 三因子信号引擎（估值·动量·基本面）
- **估值因子**：基于历史价格分位（低位=高分），`100 - percentile`
- **动量因子**：价格快照积累 + MA20/MA60 排列 + 近 20 日涨跌速率（roc20）
  - 冷启动自动从东财日线接口回补近 30 日历史收盘价，无需干等 20 天
  - 采样按交易日去重（同日仅当价格变化>0.5% 才覆盖，过滤分钟噪音）
- **基本面因子**：直接使用外部日报真实综合分，不基于百分位推算（避免循环论证）
- **加权综合评分**：估值×0.4 + 动量×0.35 + 基本面×0.25
- **趋势过滤**：低位 + 动量企稳/转多才给"买入"；低位 + 仍在下跌给"观望"（逆势不抄底）
- **弱信号**：动量未就绪时，估值极低（分位≤10%）+ 基本面不差（≥40）→ "关注"（动量待确认）
- **信号分布健康度**：买入占比>60% 时顶部警告"信号高度趋同"
- **待验证标记**：动量/基本面缺失品种标灰色"待验证"
- **免责说明**：页面顶部声明"信号仅供研究参考，不构成交易依据"

### 4. 模拟交易
- 开仓/平仓，自动计算保证金占用、风险占比、浮动盈亏
- 开仓价自动预填当前现价，品种切换联动
- 平仓后自动追加资金曲线节点

### 5. 交易日志
- 复盘记录、纪律执行率统计
- 搜索过滤栏：品种 + 类型 + 关键词三联实时过滤
- 一键"× 清空"重置筛选
- XSS 防护：全量 `escapeHtml`

### 6. 资金曲线（仪表盘）
- 累计收益、最大回撤、胜率、盈亏比、距 100 万进度
- 原生 Canvas 折线图：X 轴日期、Y 轴金额、起始资金基线、末端净值标注
- **实时点延伸**：行情刷新时自动追加当前净值点（含未平仓浮动盈亏），曲线随现价动态变化，无需等平仓
- `equityHistory` 不足时用 `closedTrades` 按日期累加构建临时曲线
- 实时点高亮显示（脉冲外圈 + "实时"标注），区别于历史已实现节点

### 7. 行情数据源（三路降级）
- **主源 futsseapi**：`futsseapi.com`（CORS 直连，覆盖 SHFE/DCE）
- **备源 东方财富 JSONP**：`push2.eastmoney.com`（覆盖 CZCE/GFEX）
- **末备 新浪期货**：`hq.sinajs.cn`（连续合约）
- 全部失败时显示具体原因（如"futsseapi 无响应 / 东财无响应 / 新浪超时"），状态灯红色
- 支持自动/手动模式切换（设置页）

### 8. GitHub Gist 云同步
- POST 创建 / PATCH 更新私有 Gist
- 恢复时本地数据更新则弹窗确认（冲突保护）
- Token 走 `sessionStorage`（关闭标签即清除），不进入 `localStorage`
- 设置页"清除 Token"按钮一键抹除

### 9. 信号通知推送
- 评级变为"买入"时触发浏览器 Notification API
- `signal.html` 通知授权按钮，授权后文案变为"已开启通知"

## 技术栈

- 纯静态 HTML + 原生 JS（无框架、无构建步骤）
- Tailwind CSS（CDN）+ 自定义设计令牌
- 原生 Canvas 2D 绘图（资金曲线，无第三方图表库）
- `localStorage` 持久化 + `sessionStorage` 安全存储
- GitHub Pages 部署
